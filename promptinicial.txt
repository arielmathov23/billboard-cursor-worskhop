PLAN — VIBE CODER BILLBOARD
Stack: Next.js (App Router)
Idioma: Español
Objetivo: construir una web simple, publicable y bien diseñada desde el primer commit

--------------------------------------------------
1. CONTEXTO GENERAL
--------------------------------------------------

Vamos a construir una web de una sola página que funcione como
un billboard personal del workshop.

No es una demo técnica.
Es una pieza de producto pequeña, cuidada y compartible.

El foco está en:
- criterio de producto
- claridad visual
- código simple y editable
- buen uso de IA como acelerador, no como magia

--------------------------------------------------
2. CONCEPTO DEL PRODUCTO
--------------------------------------------------

La página permite a cada persona:

- sacarse una foto con la cámara
- integrarla en un layout visual atractivo
- personalizar algunos elementos desde el código
- descargar una imagen final limpia (sin UI)
- quedarse con un repo propio + un commit hecho

Debe sentirse como algo que alguien
realmente podría compartir.

--------------------------------------------------
3. TEXTO Y COPY (FIJOS)
--------------------------------------------------

Título principal (exacto):
"Hoy empecé mi camino como Vibe coder"

Fecha (hardcodeada):
"22 de enero de 2026"

Nombre de la persona:
- NO editable por input
- definido como constante en el código
  (ej: const PARTICIPANT_NAME = "Tu nombre")

Todo el contenido debe estar en español.

--------------------------------------------------
4. ESTRUCTURA VISUAL DE LA PÁGINA
--------------------------------------------------

Layout vertical, centrado, con aire.

Orden:
1. Título
2. Foto circular
3. Nombre + fecha
4. Controles (cámara / descargar)
5. Footer

Diseño:
- minimalista
- moderno
- editorial
- sin ruido visual

--------------------------------------------------
5. CÁMARA (CRÍTICO)
--------------------------------------------------

Implementar cámara usando APIs nativas del browser.

Requisitos técnicos:
- usar navigator.mediaDevices.getUserMedia
- mostrar preview en vivo con <video>
- capturar imagen usando <canvas>
- relación 1:1
- imagen clara, sin glitches ni pantallas negras

IMPORTANTE (bug a evitar):
- el <video> debe existir en el DOM ANTES de asignar srcObject
- no condicionar el render del <video> a un estado posterior
- conectar el stream solo cuando el ref del video esté disponible
- manejar correctamente el lifecycle con useEffect + refs

Objetivo:
una cámara que funcione siempre, sin estados frágiles.

--------------------------------------------------
6. CAPTURA Y RESULTADO
--------------------------------------------------

Al sacar la foto:
- se congela la imagen
- se renderiza dentro de un círculo
- bien centrada
- sin distorsión
- lista para exportar

--------------------------------------------------
7. DESCARGA DE IMAGEN (NUEVO)
--------------------------------------------------

Agregar un botón “Descargar imagen”.

Requisitos:
- debe generar una imagen final
- SIN botones, inputs o UI
- solo:
  - background
  - foto
  - título
  - nombre
  - fecha

Implementación sugerida:
- usar canvas
- renderizar composición limpia
- exportar como PNG

El botón debe estar al lado de “Tomar foto”.

--------------------------------------------------
8. BACKGROUND (CLAVE DE DISEÑO)
--------------------------------------------------

El fondo define la identidad visual.

Referencia obligatoria:
- archivo en /public: 021-dithering-background.png
- debe usarse como referencia estética de dithering y patrón
- NO copiar literalmente, sino interpretar el estilo

Estilo:
- pastel
- muy sutil
- gradiente leve de bottom-left a top-right
- dominante monocromática (verde por defecto)

--------------------------------------------------
9. DITHERING / PATRÓN
--------------------------------------------------

NO usar:
- ruido random
- grain agresivo
- efectos artísticos pesados

SÍ usar:
- patrón geométrico
- cuadrados pequeños
- opacidades bajas
- variaciones mínimas de tono

Debe sentirse:
- calmado
- moderno
- editorial
- similar a visualizaciones de datos suaves

--------------------------------------------------
10. COLOR PICKER
--------------------------------------------------

Agregar un color picker simple para:
- cambiar el tono principal del background

Requisitos:
- simple
- entendible
- fácil de modificar en el código
- sin sobre-ingeniería

--------------------------------------------------
11. FOOTER
--------------------------------------------------

Footer discreto.

Contenido:
- logo fijo desde el repo: @logozerotwoone.png
- a su derecha: link "from021.io"

No debe competir con el contenido principal.

--------------------------------------------------
12. ESTILO GENERAL
--------------------------------------------------

- tipografía simple (system font o una sola fuente limpia)
- bordes suaves
- sombras mínimas o ninguna
- sin animaciones innecesarias
- diseño que “respire”

Si algo no suma al objetivo, no se agrega.

--------------------------------------------------
13. CRITERIOS DE CALIDAD
--------------------------------------------------

- todo en español
- código claro y comentado
- constantes fáciles de editar:
  - nombre
  - color principal
- evitar complejidad innecesaria
- pensar esto como un mini producto real

--------------------------------------------------
14. MENTALIDAD
--------------------------------------------------

Esto no es “hacer que funcione”.

Es:
- pensar producto
- diseñar con intención
- usar IA con criterio
- colaborar mejor con devs

Construir como si fuera a mostrarse públicamente.
